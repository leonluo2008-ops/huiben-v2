# huiben-v2 - 绘本创作智能体

> 一款专注于儿童绘本创作的 AI Agent，支持模式 A（不固定角色）和模式 B（固定角色）两种工作流，输出可直接用于即梦的生图提示词。

---

## 快速安装

```bash
# 克隆仓库
git clone https://github.com/leonluo2008-ops/huiben-v2.git
cd huiben-v2

# 或将以下文件复制到目标 Agent 工作区

# SOUL.md
```

## 工作流概览

```
步骤1: 确认摘要 + 选择模式
  ├─ 模式A（不固定角色）→ 步骤3旁白 → 步骤4路径A
  └─ 模式B（固定角色）→ 步骤2定型 → 步骤3旁白 → 步骤4路径B
```

### 步骤1：确认摘要 + 模式选择

用户输入故事主题后，Agent 复述创作意图，推荐主角/场景/风格，**主动询问用户选择模式**：

| 模式 | 说明 |
|------|------|
| **模式A（不固定角色）** | 无需参考图，角色外观在旁白中自然描述，步骤2跳过，步骤4走路径A |
| **模式B（固定角色）** | 需要上传角色参考图，步骤2生成角色定型提示词，步骤4走路径B |

### 步骤2：角色定型（仅模式B）

1. 判定参考图
2. 生成六视图定妆照提示词
3. **提示用户去即梦生成定妆照**
4. 用户带回图片后继续

### 步骤3：双语旁白创作

**规则**：
- 旁白是**故事**，不是场景描述
- 故事弧线：开头→冲突→发展→高潮→结局
- 禁止空洞句式（无互动的状态描述）
- 表格格式：`| 序号 | 英文 | 中文 |`
- 等待用户确认后进入步骤4

### 步骤4：生图提示词

- 代码块输出，代码块外无任何文字
- 路径A/B 分流
- 禁止引号包裹对话（防止渲染进图片）
- 旁白内容直接输出到聊天框，文件仅作存档

## 衍生规则（由 Darwin 优化迭代得出）

1. **EN/CN 顺序**：表格列顺序为 `| 序号 | 英文 | 中文 |`，English 在前
2. **中文提示词**：生图提示词默认使用中文描述
3. **禁止引号对话**：图片中不得出现文字，引号内文字会被即梦渲染
4. **角色变体预埋**（路径A）：同一角色的不同阶段（爸爸/宝宝）需预先定义关系
5. **故事弧线优先**：断连、跳变、千篇一律的旁白必须重写
